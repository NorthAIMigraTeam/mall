# CODEBUDDY.md

This file provides guidance to CodeBuddy Code when working with code in this repository.

## Build & Run Commands

```bash
# Build entire project (tests skipped by default via pom.xml <skipTests>true</skipTests>)
mvn clean package

# Build a single module (e.g. mall-admin)
mvn clean package -pl mall-admin -am

# Run a single service locally (requires MySQL + Redis at minimum)
cd mall-admin && mvn spring-boot:run
cd mall-portal && mvn spring-boot:run
cd mall-search && mvn spring-boot:run

# Run tests (override the default skipTests)
mvn test -DskipTests=false

# Run a single test class
mvn test -DskipTests=false -pl mall-admin -Dtest=PmsDaoTests

# Regenerate MyBatis code from database schema
cd mall-mbg && mvn mybatis-generator:generate

# Docker: start infrastructure services
docker-compose -f document/docker/docker-compose-env.yml up -d

# Docker: start application services
docker-compose -f document/docker/docker-compose-app.yml up -d

# Build Docker images via Maven (requires Docker daemon at docker.host)
mvn clean package -Pdocker docker:build
```

## Architecture

### Module Dependency Graph

```
mall-common          (Layer 0: utilities, CommonResult, Redis config, Swagger)
    |
    ├── mall-mbg     (Layer 1: MyBatis generated mappers/models + Druid + PageHelper)
    |       |
    ├── mall-security (Layer 1: Spring Security + JWT filter, depends on mall-common)
    |
    ├── mall-admin   (Layer 2: DEPLOYABLE - port 8080, depends on mall-mbg + mall-security)
    ├── mall-portal  (Layer 2: DEPLOYABLE - port 8085, depends on mall-mbg + mall-security)
    ├── mall-search  (Layer 2: DEPLOYABLE - port 8081, depends on mall-mbg only)
    └── mall-demo    (Layer 2: dev-only examples)
```

| Service | Port | Minimum Dependencies | Additional Dependencies |
|---------|------|---------------------|------------------------|
| **mall-admin** | 8080 | MySQL, Redis | MinIO/Aliyun OSS (file upload) |
| **mall-portal** | 8085 | MySQL, Redis | MongoDB, RabbitMQ, Alipay SDK |

### Package Naming Convention

All modules share the base package `com.macro.mall`. Table/entity prefixes map to business domains:

| Prefix | Domain | Example Tables |
|--------|--------|---------------|
| `Cms` | Content Management | cms_subject, cms_topic |
| `Oms` | Order Management | oms_order, oms_cart_item |
| `Pms` | Product Management | pms_product, pms_brand, pms_sku_stock |
| `Sms` | Sales/Promotion | sms_coupon, sms_flash_promotion |
| `Ums` | User Management | ums_admin, ums_member, ums_role |

### Code Organization (per service module)

```
mall-admin/src/main/java/com/macro/mall/
├── config/       # Spring configuration (Security, MyBatis, Swagger, CORS, OSS)
├── controller/   # REST endpoints (33 controllers in mall-admin)
├── dao/          # Custom MyBatis data access (hand-written SQL beyond generated code)
├── dto/          # Request/response data transfer objects
├── service/      # Business logic interfaces
│   └── impl/     # Business logic implementations
├── bo/           # Business objects (e.g., AdminUserDetails wrapping Spring Security)
└── validator/    # Custom validators
```

### Key Shared Modules

**mall-common** (`com.macro.mall.common`):
- `api/CommonResult<T>` - Unified API response wrapper. All controllers return this.
- `api/CommonPage<T>` - Paginated response wrapper (uses PageHelper).
- `api/ResultCode` - Error codes: SUCCESS(200), FAILED(500), VALIDATE_FAILED(404), UNAUTHORIZED(401), FORBIDDEN(403).
- `exception/` - Global exception handlers.
- `service/RedisService` - Redis operation wrapper.

**mall-security** (`com.macro.mall.security`):
- JWT token generation/validation with configurable secret per service.
- `JwtAuthenticationTokenFilter` - Extracts JWT from `Authorization` header.
- `RestfulAccessDeniedHandler` / `RestAuthenticationEntryPoint` - Custom 401/403 responses.
- Dynamic URL-based permission checking via `DynamicSecurityService`.

**mall-mbg** (`com.macro.mall.mapper` / `com.macro.mall.model`):
- Auto-generated by MyBatis Generator from `generatorConfig.xml`.
- ~230 mapper interfaces + corresponding XML files + model POJOs.
- Do NOT hand-edit files in this module. Re-run the generator instead.

### Configuration

- **Profiles**: `dev` (default), `prod`
- Config files: `src/main/resources/application.yml` + `application-{dev,prod}.yml`
- JWT secrets are per-service: `mall-admin-secret`, `mall-portal-secret`
- Druid connection pool monitor: accessible at `/druid/` (login: druid/druid in dev)
- Swagger UI: accessible at `/swagger-ui/` on each service

### Database

- Single MySQL database named `mall` containing all tables for all services.
- Schema + seed data: `document/sql/mall.sql` (407KB).
- Connection pool: Alibaba Druid with dev defaults (initial: 5, min-idle: 10, max-active: 20).

- `document/sh/run.sh` - Docker container bootstrap (build image, link deps, start)
- `document/sh/mall-admin.sh`, `mall-portal.sh`, `mall-search.sh` - Per-service scripts
- `document/sh/Dockerfile` - Generic Java 8 image template

### Key Patterns

1. **API Response**: Always use `CommonResult.success(data)` / `CommonResult.failed(msg)` in controllers.
2. **Pagination**: Use `PageHelper.startPage(pageNum, pageSize)` before the query, then wrap with `CommonPage.restPage(list)`.
3. **Authentication**: JWT token in `Authorization: Bearer <token>` header. Login endpoints are whitelisted in `secure.ignored.urls` config.
4. **Custom SQL**: Place hand-written queries in the service module's `dao/` package with corresponding `src/main/resources/dao/*.xml` mapper XML files. Keep generated code in mall-mbg untouched.
5. **Redis caching**: Key patterns defined in `application.yml` per service (e.g., `ums:admin`, `ums:resourceList`).
6. **File upload**: Mall-admin supports both Aliyun OSS and MinIO. Configuration is in `OssConfig.java` / `MinioController.java`.
7. **Message queue**: RabbitMQ used in mall-portal for async order cancellation via `cancelOrderQueue`.
